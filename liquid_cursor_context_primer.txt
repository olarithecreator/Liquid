You are helping me build Liquid — a Nigerian digital asset platform
with two independent engines:

1. USDT EXCHANGE
Users buy and sell USDT against Naira via a manual P2P process.
- Buy: user pays Naira to our bank account, we send USDT to their wallet
- Sell: user sends USDT to our wallet, we pay Naira to their bank
- Every order has a 20-minute payment window before it auto-expires
- Admin verifies all transactions manually via a web dashboard
- No automated blockchain integration — fully manual operator execution

2. PORTFOLIO INTELLIGENCE (subscription)
Admin publishes market insights. Users subscribe to read them.
- Three tiers: Basic (₦15,000/mo), Business (₦45,000/mo), Institutional (₦120,000/mo)
- Posts are tier-locked — each post has a minimum tier to read it
- Payment via Paystack inline popup
- Completely independent from the exchange — users can subscribe without
  ever doing an exchange, and can exchange without subscribing

EXCHANGE:
- Orders expire exactly 20 minutes after creation (pg_cron runs every minute)
- Timer starts only after the order is created in DB — not when screen loads
- Timer colour: gold (20:00–10:01) → orange (10:00–3:01) → red+pulse (3:00–0:00)
- Buy complete:  "Your USDT has been sent to your wallet"
- Sell complete: "₦[amount] has been sent to your [bank] account ••[last4]"
- 06B state: when status = awaiting_payment and no proof uploaded yet,
  show upload zone directly on the Order Status screen
- Supabase realtime subscription watches orders table —
  status changes appear instantly without user refreshing

EXCHANGE STATES:
- exchange_open = true  → normal home, cards fully visible
- exchange_open = false → red status bar, action cards at 60% opacity,
  tapping either card goes to /exchange/closed not /exchange/buy

INTELLIGENCE / SUBSCRIPTION:
- tier lock function:
    canView('all', anyPlan)          → true
    canView('basic', plan)           → basic, business, institutional
    canView('business', plan)        → business, institutional only
    canView('institutional', plan)   → institutional only
- Locked post: blurred body, lock overlay, "Upgrade" button → /plans
- No-plan users: gold subscription prompt banner above feed
- Plan badge in top-right of feed header — tapping → /plans
- Subscription is via Paystack only — not manual

ADMIN:
- Admin email is set in VITE_ADMIN_EMAIL environment variable
- Sign in at /admin/login (separate from /signin)
- If someone signs in with a non-admin email → sign them out immediately,
  show "Access denied" toast, stay on /admin/login
- Visiting /admin/* without admin auth → redirect to /admin/login
- Pending orders badge on sidebar updates in realtime

NOTIFICATIONS:
- Every order created → Telegram alert to admin + email to user
- Every proof uploaded → Telegram alert to admin
- Order completed → email to user + in-app notification
- New subscriber → Telegram alert to admin + in-app notification to user
- Unread notifications show badge on the portfolio nav icon (▦)
