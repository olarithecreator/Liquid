Section 01
App Launch & Routing
The very first thing the app does when it opens. These decisions happen before any screen renders.
⚡

Check two flags: liquid_onboarded and the Supabase auth session token.
◆
Is user logged in? (Supabase session exists)
YES → Check if email === ADMIN_EMAIL. If yes → → /admin Dashboard. If no → → Home
NO → Check if liquid_onboarded === 'true'. If yes → → Sign In. If no → → Splash
⚡
On Home load — fetch live data
Immediately fetch: (1) app_settings row buy_rate, (2) user's last 4 orders, (3) user's portfolio_value. Show skeleton loading state while fetching. Never show stale hardcoded data.
◆

Fetch app_settings key='announcement'. If value is not empty → show gold banner at top of Home. If empty → no banner. This banner appears above everything else on Home.

Section 05
Exchange Flow — The 20-Minute Timer
The timer is the most complex UX in the app. It has multiple visual states, a database expiry, and affects what the user can do. Read this section carefully.
⏱ Timer — Complete Rules
When does the timer START? The moment the user taps "Confirm Order" or "Submit for Verification" and the order is successfully created in the database. Not when they open the screen — only when they commit.
Timer display: MM:SS countdown. Colour changes: 20:00–10:01 = gold, 10:00–3:01 = orange (#D97706), 3:00–0:01 = red with pulsing animation. At 0:00 = expired.
Progress bar: Depletes from 100% to 0% over 20 minutes. Matches the colour changes above.
Timer on the Order Status screen: NOT shown here. The Order Status screen shows the step tracker instead. The timer only shows on the Acquire and Sell Step 2 screens.
What happens at 0:00 on the client? If the order status is still 'awaiting_payment' (no proof uploaded), automatically navigate to → Order Expired screen. If proof was already uploaded (status = proof_uploaded), the timer disappears and shows "Proof received — waiting for admin" message instead.
Database expiry: A Supabase function runs every minute. Any order with status = 'awaiting_payment' older than 20 minutes is updated to status = 'expired'. This is the source of truth — the client timer is visual only.
Timer persistence: If the user leaves the app and comes back, the timer resumes from the correct time. Calculate remaining time as: created_at + 20 minutes - now(). If result is ≤ 0 → order is already expired.
Timer on Acquire screen (before order creation): NOT shown. The timer block only appears AFTER the order is created, on the Order Status screen. The Acquire/Sell screens show the timer ONLY when returning to a screen with an existing active order.

Section 09
Global UX Rules for Cursor
These apply to every screen in both apps. Give this list to Cursor at the start of every build session.
GlobalLoading States
Any button that makes an API call
Show a loading spinner inside the button. Disable the button (not tappable). Revert to normal after API responds (success or error). Never leave a button in a permanently loading state.
Any screen that fetches data on load
Show skeleton loading blocks (grey animated placeholders) where content will appear. Same dimensions as real content. Replace with real data when fetch completes. Never show a blank white screen.
Fetch error (network down)
Show a retry state: icon, "Couldn't load data", grey "Try Again" button. Log error to console. Do not crash the app.
GlobalToast Notifications (in-app)
Success
Green toast at top of screen. Shows for 3 seconds then auto-dismisses. Has ✓ icon. Example: "Rate updated successfully"
Error
Red toast at top. Shows for 4 seconds (slightly longer so user can read it). Has ✕ icon. Example: "Payment failed. Try again."
Info
Gold toast at top. Shows for 3 seconds. Has ℹ icon. Example: "Order copied to clipboard"
Multiple toasts
Stack toasts vertically. New ones appear below existing ones. Each dismisses independently.
GlobalBack Navigation Rules
Screens with back arrow
Acquire, Sell Step 1, Sell Step 2, Order Status, Exchange Closed, Plans, Transaction History, Notifications, Account Settings. All go back to their logical parent screen.
Screens WITHOUT back arrow
Home, Intelligence, Portfolio — these are tab-root screens. No back arrow. Only accessible via nav tabs.
Android hardware back button
Same behaviour as tapping the on-screen back arrow. If on a root tab screen with no back history → do nothing (do not close the app).
Sell Step 2 back button
Goes to Sell Step 1 (not Home). Bank details from Step 1 must still be in state — user should not have to re-enter them.
GlobalKeyboard Behaviour
Keyboard opens on input focus
Screen content scrolls up so the focused input is visible above the keyboard. Content below the keyboard does not get cut off — the screen is scrollable while keyboard is open.
Tap outside input
Keyboard dismisses. No other action.
USDT amount field type
Keyboard type = numeric (only numbers, no letters). inputmode="decimal" to allow decimal entry.
Account number field type
Keyboard type = numeric. maxLength = 10. No decimal allowed.
GlobalEmpty States
No orders yet (Activity on Home)
Show illustration + "No transactions yet" + "Your first order will appear here". Do not show empty rows.
No notifications
Show "You're all caught up" + "New notifications will appear here"
No intelligence posts published yet
Show "Intelligence posts coming soon" message. Do not show empty card shapes.
Admin Orders page — no pending orders
Show a green "✓ All clear — no pending orders" state. Feels reassuring not alarming.
