◉ Setup
Phase 00
Install Tools & Create Project
Do this once. Every command runs in your computer's Terminal (Mac) or Command Prompt (Windows).
1
Install Node.js

Go to nodejs.org → download the LTS version → install it. This is what runs JavaScript on your computer. After installing, open Terminal and verify:

Terminal — verify installation
node --version
# Should print: v20.x.x or higher

npm --version
# Should print: 10.x.x or higher
2
Install Cursor

Go to cursor.com → download → install. Open Cursor. Sign in (free account is fine to start). Cursor is VS Code with AI built in — the AI can read and write your entire codebase at once.

3
Create the project

In Terminal, navigate to where you want to create the project folder, then run:

Terminal — create project
# Navigate to your projects folder (example)
cd ~/Desktop

# Create the React + TypeScript + Vite project
npm create vite@latest liquid-app -- --template react-ts

# Move into the new folder
cd liquid-app

# Install base dependencies
npm install

# Install all the libraries Liquid needs
npm install @supabase/supabase-js react-router-dom @tanstack/react-query
npm install lucide-react clsx date-fns
npm install -D tailwindcss postcss autoprefixer @types/node

# Set up Tailwind CSS
npx tailwindcss init -p
4
Open project in Cursor

In Terminal, while inside the liquid-app folder:

Terminal
cursor .

This opens the whole project in Cursor. You will see all your files on the left sidebar. From now on, you work inside Cursor — not in the Terminal except when instructed.

5
Configure Tailwind + environment file

In Cursor, open the file tailwind.config.js and replace everything with:

tailwind.config.js
export default {
  content: ['./index.html', './src/**/*.{js,ts,jsx,tsx}'],
  theme: {
    extend: {
      colors: {
        gold: '#C8973F',
        'gold-light': '#E8B865',
        ink: '#0C0C0C',
        'ink-2': '#1A1A1A',
      },
      fontFamily: {
        heading: ['Syne', 'sans-serif'],
        body: ['Inter', 'sans-serif'],
      },
    },
  },
  plugins: [],
}

Now create a file called .env.local in the root of your project (same level as package.json). This file stores your secret keys — it is never shared or uploaded to GitHub:

.env.local — create this file
VITE_SUPABASE_URL=your_supabase_url_here
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key_here
[email protected]
VITE_PAYSTACK_PUBLIC_KEY=pk_test_xxxx
VITE_RESEND_API_KEY=re_xxxx
⚠️
Never commit .env.local to GitHub. Cursor automatically adds it to .gitignore — verify this is the case. Your real keys must never appear in your code files.
6
Test that the project runs
Cursor Terminal (use the terminal inside Cursor: View → Terminal)
npm run dev

Open your browser and go to http://localhost:5173. You should see a basic Vite + React page. If you see it → setup is working. Press Ctrl+C in terminal to stop it.

Phase 02
Supabase — MCP Direct Connection
Cursor connects directly to your Supabase project through MCP. Once connected, you type plain English and Cursor creates every table, every policy, every trigger — no copy-pasting SQL, no switching between apps. This phase has four parts: create your Supabase account, get your API keys, connect MCP to Cursor, then let Cursor build the entire database.
Part A — Create your Supabase account (do this in your browser)
1
Sign up at supabase.com

Open your browser → go to supabase.com → click "Start your project" → click "Continue with GitHub". Log in to GitHub and click Authorize. This links both accounts — no separate passwords needed.

ℹ️
No GitHub yet? Go to github.com first → sign up free → then come back to supabase.com and sign up with GitHub.
2
Create your project

After signing in, click "New project". Fill in exactly:

Project name: liquid-app

Database password: click "Generate a password" → copy it and save it in your phone Notes app. You will need it later.

Region: West EU (Ireland) — fastest connection from Nigeria.

Plan: Free.

Click "Create new project". Wait 2 minutes while Supabase sets up. You'll see a loading screen then your project dashboard.

3
Copy your API keys into .env.local

In your Supabase project → click Settings (gear icon, bottom of left sidebar) → click API. You will see two values. Copy both:

Project URL — looks like: https://abcdefgh.supabase.co

anon public key — a long string starting with eyJ

Now open your .env.local file in Cursor and fill them in:

.env.local — paste your real values
VITE_SUPABASE_URL=https://your-project-id.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGc...your-full-key-here

Never share .env.local. Never post these values anywhere. Never push this file to GitHub. These keys give full access to your database.
Part B — Get your Supabase MCP Access Token

This is a special key that lets Cursor talk to Supabase directly. It is different from your API key. You only need to do this once.

1
Go to your Supabase account settings

In the Supabase dashboard → click your profile picture or avatar in the very top right corner of the page → click "Account" from the dropdown menu. This takes you to your personal account settings, not the project settings.

2
Generate a new access token

In Account settings → look in the left sidebar for "Access Tokens" → click it → click the "Generate new token" button → in the name field type cursor-liquid → click Generate token.

A long token appears on screen. Copy it immediately — this is the only time you will ever see it. Once you close this dialog it is gone forever. If you lose it, you generate a new one.

⚠️
Copy the token RIGHT NOW before closing the dialog. Paste it into your Notes app immediately. You cannot view it again after closing.
Part C — Connect Supabase MCP inside Cursor

You do this once. After this, Cursor can read and write your database directly whenever you ask it to.

1
Open Cursor Settings

On Mac: In the top menu bar click Cursor → click Settings → click Cursor Settings.

On Windows: Click File in the top menu → click Preferences → click Cursor Settings.

A settings panel opens on the right side of Cursor.

2
Find the MCP section

In the settings panel, scroll down until you see a section called "MCP" or "Model Context Protocol". Click "Add new MCP server" or click "Edit MCP config" — either option works, both open the same JSON config file.

3
Paste the MCP configuration

A file called mcp.json opens. It might be empty or have existing content. Replace everything in it with this (paste your actual Supabase access token where shown):

mcp.json — paste this entire block
{
  "mcpServers": {
    "supabase": {
      "command": "npx",
      "args": [
        "-y",
        "@supabase/mcp-server-supabase@latest",
        "--access-token",
        "PASTE_YOUR_SUPABASE_ACCESS_TOKEN_HERE"
      ]
    }
  }
}

Replace PASTE_YOUR_SUPABASE_ACCESS_TOKEN_HERE with the token you copied in Part B. Save the file (Cmd+S or Ctrl+S).

⚠️
The token goes inside the quotes. The final line should look like: "sbp_abc123yourtokenhere" — with the token inside the quotes, replacing the placeholder text.
4
Verify the connection works

After saving mcp.json, Cursor restarts its MCP connection automatically. To confirm it worked: press Cmd+I (Mac) or Ctrl+I (Windows) to open Cursor Composer. In the bottom of the Composer panel look for a tools icon or a "supabase" label — it should show as connected.

Test it by typing this in Composer and pressing Enter:

Test prompt — verify MCP is working
Using the Supabase MCP tool, list all projects in my Supabase account 
and tell me the name and ID of the liquid-app project.

Cursor will call Supabase directly and show you your project details. If you see your liquid-app project listed — MCP is connected and working. If you see an error, go back to Part B, generate a new token, and replace it in mcp.json.

✓
Connection confirmed. From this point on, every time you ask Cursor to do something with your database, it does it directly — no switching to the Supabase website, no copy-pasting SQL.

━━━ UX RULES ━━━
- Timer is the source of truth for the user's payment window
- If the user closes the app and reopens: timer resumes from correct time
  (it calculates from created_at + 20min - now(), not from a stored countdown)
- When timer hits 0:00 — call onExpire() immediately
- onExpire() should: navigate to /exchange/expired/[orderId]
- If proof is uploaded before timer hits 0: parent hides the timer component
- Do not use setInterval alone — always derive time from created_at timestamp
  so the timer stays accurate even if the tab was in the background

Cursor Composer Prompt 11-A — Supabase Edge Functions
Create two Supabase Edge Functions to handle notifications.
These run server-side so API keys are never exposed to the browser.

First, install the Supabase CLI in Terminal:
npm install supabase --save-dev
npx supabase init
npx supabase login

Then create the edge functions:
npx supabase functions new send-email
npx supabase functions new send-telegram

═══ supabase/functions/send-email/index.ts ═══
HTTP POST endpoint that accepts:
{ to: string, subject: string, body: string }

Uses Resend API:
const res = await fetch('https://api.resend.com/emails', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${Deno.env.get('RESEND_API_KEY')}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    from: 'Liquid ',
    to: [to],
    subject: subject,
    html: body
  })
})

Email templates to support (passed as 'type' in request body):
- 'order_created': Order confirmation with payment details
- 'proof_received': Proof received, verification in progress
- 'order_completed': Order done, USDT/Naira sent
- 'subscription_activated': Welcome to [plan]

═══ supabase/functions/send-telegram/index.ts ═══
HTTP POST endpoint that accepts: { message: string }

Reads token and chat_id from app_settings table.
Sends to Telegram Bot API:
await fetch(`https://api.telegram.org/bot${token}/sendMessage`, {
  method: 'POST',
  body: JSON.stringify({ chat_id: chatId, text: message, parse_mode: 'HTML' })
})

Deploy both functions:
npx supabase functions deploy send-email
npx supabase functions deploy send-telegram

Set secrets in Supabase dashboard → Edge Functions → Manage secrets:
RESEND_API_KEY = your_resend_key
